Sintaxis Básica

Limón es un lenguaje basado en expresiones sencillas, diseñado para ser fácil de leer y escribir. A diferencia de muchos lenguajes tradicionales que pueden requerir una sintaxis compleja y detallada, Limón apuesta por la simplicidad en su estructura. Esto lo convierte en una opción ideal para principiantes o para proyectos donde la legibilidad del código es primordial. Al eliminar los elementos superfluos, Limón permite a los desarrolladores concentrarse en la lógica y la funcionalidad del programa, en lugar de en los detalles técnicos que a menudo complican otros lenguajes.

La sintaxis de Limón es minimalista, enfocada en reducir la complejidad innecesaria. Cada instrucción sigue un formato claro y directo, sin la necesidad de símbolos o estructuras que dificulten la comprensión. Por ejemplo, las declaraciones de variables y funciones en Limón utilizan palabras clave intuitivas que reflejan el propósito del código, facilitando su lectura tanto para el desarrollador como para cualquier colaborador. Esta filosofía de diseño busca no solo optimizar el proceso de programación, sino también hacer que el código sea fácilmente mantenible y adaptable a cambios futuros.

definir variable = 10; definir nombre = "Juan"; definir esActivo = verdadero; definir resultado = sumar(5, 7); definir lista = [1, 2, 3, 4];
  • Las declaraciones siempre terminan con punto y coma, siguiendo una convención clara y consistente en Limón. Este símbolo no solo marca el final de una instrucción, sino que también contribuye a la legibilidad del código y evita posibles ambigüedades durante la ejecución. Por ejemplo, en las declaraciones como definir variable = 10; o definir lista = [1, 2, 3, 4];, el punto y coma garantiza que el intérprete de Limón entienda correctamente cuándo ha finalizado una instrucción. Esta regla uniforme también ayuda a evitar errores comunes, como la fusión accidental de múltiples expresiones, algo que puede suceder en lenguajes que omiten este símbolo.
Tipos de Datos

Limón soporta varios tipos de datos primitivos como números, cadenas, y booleanos. Estos tipos de datos básicos permiten a los desarrolladores almacenar y manipular información fundamental de manera eficiente. Los números en Limón pueden ser enteros o decimales, lo que facilita tanto cálculos simples como operaciones más precisas. Las cadenas permiten almacenar texto, mientras que los booleanos representan valores lógicos como verdadero o falso. Estos tres tipos de datos son la base de muchas operaciones comunes en la programación y son esenciales para la construcción de cualquier lógica en Limón.

cadena nombre = "Limón";

Los tipos de datos pueden ser explícitamente declarados o inferidos por el compilador. Esto significa que el desarrollador puede optar por especificar el tipo de cada variable, como en el ejemplo anterior con cadena, o bien dejar que el compilador de Limón deduzca el tipo de dato a partir del valor asignado. Esta flexibilidad permite una mayor comodidad durante el desarrollo, ya que en algunos casos se puede escribir código más conciso sin sacrificar la seguridad en los tipos. Limón equilibra la simplicidad con el control, dándole al programador la opción de elegir entre la inferencia automática o la declaración explícita de tipos.

  • Números enteros y decimales están soportados, permitiendo operaciones matemáticas tanto simples como avanzadas. Un número entero, como definir edad = 25;, es útil para contar elementos, mientras que un número decimal, como definir precio = 19.99;, es ideal para cálculos que requieren mayor precisión, como aquellos involucrados en operaciones financieras o científicas.
Estructuras de Control

Limón cuenta con estructuras de control como condicionales y bucles para facilitar la toma de decisiones y la repetición de tareas. Las condicionales permiten que el programa evalúe ciertas condiciones y ejecute bloques de código en consecuencia, lo que es esencial para adaptarse a diferentes situaciones durante la ejecución del programa. Por ejemplo, una estructura condicional puede verificar si una variable cumple con un valor específico antes de ejecutar una acción, como en si (condicion) { ejecutar_algo(); }. Esto le da al lenguaje la flexibilidad necesaria para manejar comportamientos dinámicos y reacciones basadas en la lógica de la aplicación.

El lenguaje también soporta estructuras de control avanzadas como bucles "mientras", que permiten repetir una sección de código mientras una condición sea verdadera. Los bucles son fundamentales para iterar sobre colecciones de datos o ejecutar tareas repetitivas sin duplicar código. Por ejemplo, un bucle "mientras" en Limón podría verificar constantemente si un valor es menor que cierto límite, ejecutando una acción en cada iteración. Estas estructuras avanzadas proporcionan las herramientas necesarias para controlar el flujo del programa de manera eficiente y limpia, permitiendo la automatización de tareas repetitivas sin necesidad de escribir múltiples veces la misma instrucción.

  • Las condiciones se definen con operadores lógicos, como y, o, y no, que permiten evaluar combinaciones de condiciones. Por ejemplo, si (a es mayor que 5 y b es menor que 10) { hacer_algo(); } ejecutaría el bloque de código solo si ambas condiciones son verdaderas. Estos operadores permiten construir expresiones complejas que pueden influir en las decisiones del programa de manera más precisa y detallada.
Funciones

Las funciones en Limón permiten organizar el código en bloques reutilizables, facilitando la modularidad y la claridad en los programas. Al encapsular una serie de instrucciones dentro de una función, los desarrolladores pueden ejecutar esa lógica en cualquier parte del código simplemente llamando a la función por su nombre. Esto no solo evita la duplicación de código, sino que también facilita la lectura y el mantenimiento de los programas. Por ejemplo, la función saludar se puede invocar múltiples veces para realizar la misma tarea, en lugar de escribir la instrucción imprimir("Hola Mundo!"); cada vez que se necesite.

función saludar() { imprimir("Hola Mundo!"); }

Las funciones pueden recibir parámetros y devolver valores, lo que aumenta su flexibilidad y capacidad para interactuar con diferentes partes del programa. Al aceptar parámetros, una función en Limón puede adaptarse a diferentes situaciones sin cambiar su definición básica. Por ejemplo, una función de suma podría aceptar dos números como parámetros y devolver su resultado. Esto permite que las funciones sean más dinámicas y útiles en una variedad de contextos. Además, el valor de retorno proporciona una forma de comunicar el resultado de la función a otras partes del programa, facilitando cálculos y la toma de decisiones en base a esos resultados.

  • Las funciones se definen con la palabra clave función, seguida del nombre de la función y una lista de parámetros opcionales. Por ejemplo, función sumar(a, b) { devolver a + b; } define una función que toma dos parámetros, a y b, y devuelve su suma. Esta sintaxis simple y directa facilita la creación de funciones de manera intuitiva, permitiendo que los desarrolladores escriban código de manera más rápida y sin complicaciones innecesarias.
Manejo de Errores

Limón implementa un sistema sencillo para manejar errores a través de bloques de captura, permitiendo a los desarrolladores gestionar excepciones de manera efectiva. Este enfoque se basa en el uso de bloques tratar para envolver el código que podría generar un error y bloques capturar para definir cómo se debe manejar ese error si ocurre. Este sistema facilita el control del flujo del programa, asegurando que cualquier situación inesperada se aborde de manera ordenada, en lugar de provocar una falla abrupta del programa. Por ejemplo, en el bloque tratar { ejecutar(); }, se intenta ejecutar la función ejecutar, y si se produce un error durante esta ejecución, el control se transfiere automáticamente al bloque capturar (Error) { manejar_error(); }, donde se define la respuesta al error.

tratar { ejecutar(); } capturar (Error) { manejar_error(); }

El manejo de errores es fundamental para asegurar que el programa se ejecute correctamente bajo diversas condiciones. Sin un manejo adecuado, los errores pueden llevar a resultados inesperados o, en el peor de los casos, a la interrupción completa del programa. Limón permite a los desarrolladores anticiparse a posibles fallos y proporcionar soluciones adecuadas, mejorando la robustez y la confiabilidad del código. Este enfoque también facilita la depuración, ya que permite a los programadores capturar y registrar información sobre errores específicos, lo que a su vez les ayuda a identificar y corregir problemas de manera más efectiva.

  • Los errores se manejan con bloques tratar y capturar, que permiten una separación clara entre el código que podría fallar y la lógica para manejar esos fallos. Por ejemplo, al utilizar tratar, los desarrolladores pueden incluir múltiples instrucciones que podrían generar excepciones, y mediante capturar, especificar cómo responder ante diferentes tipos de errores, lo que aporta flexibilidad y control al proceso de desarrollo. Esta estructura no solo mejora la calidad del software, sino que también proporciona una experiencia más suave para los usuarios finales.
